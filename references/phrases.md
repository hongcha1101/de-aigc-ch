# 必须删除的词句

## 废话开场

删掉这些空转开头。直接说核心内容。

> 这是中文 AI 最高频的词汇层特征之一。"值得注意的是""众所周知"等废话开头，是判别 AI 文本的核心标志。

- "值得注意的是，"
- "不可否认的是，"
- "毋庸置疑的是，"
- "众所周知，"
- "值得一提的是，"
- "首先需要明确的是，"
- "说到底，"
- "归根结底，"
- "从某种程度上来说，"
- "从某种意义上说，"
- "平心而论，"
- "客观来说，"
- "坦白说，"
- "严格来说，"
- "不得不承认的是，"
- "需要指出的是，"
- "必须强调的是，"
- "不容忽视的是，"
- "需要说明的是，"
- "值得一提的是，"
- "令人关注的是，"
- "更不用说——"

所有"值得……的是""需要……的是"结构都是废话铺垫。砍掉。

**替换原则**: 直接说你要说的内容。把"值得注意的是，这个方案有三个优点"改为"这个方案有三个优点"。

### 递进链——"不只是……更是……" 及其变体

> ⚠️ **最高频 AI 句式警告**：这是目前中文 AI 文本中最容易被识别的句式特征 [1][2]。

这是目前中文 AI 文本中最容易被识别的句式特征。"不只是……更是……"及其变体的出现频率极高 [1][2]。

| 句式 | 出现频率 | AI 指数 |
|------|---------|---------|
| 不只是……更是…… | 极高 | ⭐⭐⭐⭐⭐ |
| 不仅是……更是…… | 极高 | ⭐⭐⭐⭐⭐ |
| 不仅仅是……更重要的是…… | 高 | ⭐⭐⭐⭐ |
| 不单是……而且是…… | 中 | ⭐⭐⭐ |
| 不只是……同时也是…… | 高 | ⭐⭐⭐⭐ |
| 不仅是……也是…… | 高 | ⭐⭐⭐⭐ |
| 并非……而是…… | 中 | ⭐⭐⭐ |
| 不是……而是…… | 中（但近年飙升） | ⭐⭐⭐⭐ |
| 既是……也是……更是……（三重递进） | 中 | ⭐⭐⭐⭐⭐ |

**"不是……而是……"**: 虎嗅网 2026 年度回顾文章称 2025 年是"'不是……而是……'句式刷屏的一年"，
大量自媒体和 AI 生成内容密集使用此结构进行"伪对比"。

**"既是……也是……更是……"**: 这是 AI 特有的"三重递进"结构。正常人类最多用两层，AI 喜欢用到三层。
看到三个"是"连续递进，几乎可以判定为 AI 写作。

**替换策略**:
1. 把递进改为直接并列：将"这不只是效率提升，更是工作方式的革命"改为"这提升了效率，也改变了工作方式"
2. 完全删除递进关系：直接用事实说话，不需要用递进结构来"强调重要性"
3. 使用更自然的连接词：用"而且"、"同时"、"还"代替"不只是……更是……"

## 泛化引用

> ⚠️ **极高风险句式** —— 不仅容易被 AI 检测抓出，还可能导致事实性错误。

在 AI 生成的学术文本中，大量引用为完全虚构的幻觉引用（不存在于任何真实文献中）。
这些引用往往以"泛化引用"句式开头，看起来像在引用研究，实际是 AI 凭空编造。

| 句式 | 风险等级 | 说明 |
|------|---------|------|
| "研究表明……" | ⭐⭐⭐⭐⭐ | 最高频幻引句式，AI 极可能编造研究 |
| "有学者指出……" | ⭐⭐⭐⭐ | 无法验证，99% 无具体出处 |
| "据调查显示……" | ⭐⭐⭐⭐⭐ | 经常伴有编造的统计数据 |
| "大量研究表明……" | ⭐⭐⭐⭐⭐ | "大量"二字是幻引的遮羞布 |
| "研究证实……" | ⭐⭐⭐⭐ | 暗示存在具体实验但实际没有 |
| "根据相关研究……" | ⭐⭐⭐⭐ | "相关"二字让引用无法追查 |
| "国外学者发现……" | ⭐⭐⭐⭐⭐ | 利用信息差制造虚假权威 |
| "统计数据表明……" | ⭐⭐⭐⭐ | 经常伴随编造的具体数字 |
| "近年来研究发现……" | ⭐⭐⭐⭐ | 时间范围模糊，无法查证 |
| "业内专家认为……" | ⭐⭐⭐⭐ | "专家"是匿名权威的经典用法 |
| "据权威机构统计……" | ⭐⭐⭐⭐⭐ | "权威机构"从不说出具体名称 |
| "历史经验表明……" | ⭐⭐⭐ | 相对安全但仍是空洞引用 |

**替换策略**:
1. **必须加具体出处**: 写清楚作者名加年份加期刊，有真实来源可查。
2. **或者删掉整句**: 如果找不到具体出处，直接删除
3. **转化为个人观点**: "我认为……"比"研究表明……"更诚实
4. **检查真实性**: 如果引用了具体研究，必须确认该研究真实存在

> **黄金法则**: 凡是你写不出作者+年份+期刊的"引用"，全部删除。

## 冒号/破折号过度使用

> 对比分析显示：AI 文本中冒号使用频率是人工写作的 3.2 倍，破折号（——）使用频率是人工写作的 2.7 倍 [3]。

### 冒号滥用

AI 喜欢在每段开头用冒号引出解释或列表：

- "这意味着："
- "具体来说："
- "主要体现在以下几个方面："
- "原因如下："
- "主要包括："
- "具体包括："
- "特点如下："
- "具体表现为："
- "概括起来有以下几点："
- "一是……；二是……；三是……"

**替换策略**: 把冒号后的内容直接写成正常的句子，不需要用冒号"预告"。

> ❌ AI 写法: "这个方案有三个优点：一是成本低，二是效率高，三是易维护。"
> ✅ 人工写法: "这个方案成本低、效率高、易维护。"

> ❌ AI 写法: "人工智能的发展面临几个挑战：数据隐私问题、算法偏见问题、监管问题。"
> ✅ 人工写法: "人工智能在数据隐私、算法偏见和监管方面面临挑战。"

### 破折号滥用

AI 喜欢用破折号表示强调，一篇文章出现 3 次以上就是 AI 特征：

- "这是一个——关键的——转折点。"
- "我们需要做出——正确的——选择。"
- "这不仅仅是一个问题——更是一个警示。"
- "最终的结果是——失败了。"
- "答案只有一个——那就是——"
- "这个方案的核心——也是它的最大优势——在于——"

**替换策略**: 一篇 2000 字文章中出现超过 2 个破折号，就需要警惕。用逗号或句号替换，或者直接重写句子。

> ❌ AI 写法: "这个决定——虽然充满争议——却是正确的——也是必要的。"
> ✅ 人工写法: "这个决定虽然充满争议，但是正确且必要的。"

## 空洞强调

想说"这个很重要"但没给出任何事实依据。

> 绝对化断言是 AI 文本的另一个检测特征。AI 倾向于用强调词来弥补论证的不足——因为没有具体论据，所以用"极其重要"来刷存在感。

- "非常重要"
- "极其关键"
- "毋庸置疑"
- "显而易见"
- "毫无疑问"
- "不言而喻"
- "众所周知"
- "不可忽视的是"
- "尤其需要注意的是"
- "最值得注意的是"
- "更值得关注的是"
- "极为重要"
- "至关重要"
- "不可或缺"
- "不可替代"
- "举足轻重"
- "至关重要的一环"

如果一句话说了"重要/关键/值得关注"但没有给出"为什么"，删掉或补充理由。

> **替换测试**: 把句子里的强调词去掉，如果句子依然成立（或更有力），说明强调词是多余的。
> 如果去掉后句子不成立，说明你本来就没写出理由，需要补充论据。

## 模板化开头段落

> 这类"时间背景+宏大叙事+引出主题"的开头模式，在 AI 生成的中文文本中很常见。人类作者极少用这种"三段式"开头。

### 近年/随着 开头

- "随着……的……，……日益……"
- "在……的背景下，……成为了……"
- "近年来，……越来越……"
- "在当前……的大环境下，……"
- "随着时代的不断发展，……"
- "在……快速发展的今天，……"
- "伴随着……的深入，……"
- "在……的浪潮中，……"
- "当今世界正经历百年未有之大变局，……"
- "进入……时代，……"
- "随着……的普及，……"
- "在……的推动下，……"

**替换策略**: 直接把整个开头段落删掉，从主语开始写。

> ❌ AI 写法: "随着互联网技术的飞速发展，人工智能技术日益成熟，在各个领域得到了广泛应用。"
> ✅ 人工写法: "人工智能已经用在医疗和教育领域。"

> ❌ AI 写法: "在数字化转型的大背景下，企业面临着前所未有的挑战和机遇。"
> ✅ 人工写法: "不转型的企业正在失去市场。"

### "……是……的必然要求" 结构

- "是……的必然选择"
- "是……的必由之路"
- "是……的客观要求"
- "是……的内在需要"
- "是……的应有之义"
- "是……的根本保障"
- "是……的重要前提"
- "是……的关键所在"
- "是……的题中应有之义"

**替换策略**: 直接说"需要做什么"，不要套这个空壳子。

> ❌ AI 写法: "加强数据安全是企业数字化转型的必然要求。"
> ✅ 人工写法: "企业需要加强数据安全。"

## 语气软化剂

AI 不敢把话说死，所以加一堆软化词。删掉它们。

> AI 文本中的模糊限制语密度是人类文本的 2-4 倍 [1]。连续使用两个以上软化词是 AI 文本的显著特征。

- "可能"
- "或许"
- "大概"
- "某种程度上"
- "在一定程度上"
- "相对而言"
- "比较"
- "有些"
- "一定的"
- "一定的程度上"
- "有可能会"
- "可以说是"
- "可以说是比较重要"
- "或许可以说是"
- "似乎"
- "看起来"
- "往往"
- "通常"
- "一般说来"
- "某种程度上说"
- "在某种意义上"
- "某种程度"
- "可以认为"
- "大致"
- "有一定程度的"

连续两个以上软化词同时出现，整句重写。

> ❌ AI 写法: "这个方案可能会在一定程度上对效率有所提升。"
> ✅ 人工写法: "这个方案能提升效率。" —— 或者 —— "这个方案不能保证效率提升。"

**原则**: 要么说确切，要么保持沉默。用"可能大概也许"来回避责任不是严谨，是懦弱。

## AI 常见尾气

- "以上是关于……的全部内容。"
- "希望以上内容对你有所帮助。"
- "如果你有任何问题，欢迎随时交流。"
- "如果你有其他想法，也欢迎一起来探讨。"
- "总的来说，这是一个值得持续关注的话题。"
- "希望本文能为你提供一些参考。"
- "感谢你的阅读。"
- "如果你觉得有用，请点赞收藏。"
- "欢迎在评论区留言讨论。"
- "以上就是我的分享。"
- "希望能对你有所启发。"
- "谢谢大家。"
- "下面是总结时间。"

**替换策略**: 直接结束。不需要客套，不需要总结，不需要邀请讨论。

> **原则**: 文章的结尾是最后一个句号。不需要加"售后"。

## 元评论

提醒读者"我在写文章"的废话。

- "正如前文所述，"
- "如上所述，"
- "相关内容将在后续章节中详细讨论。"
- "限于篇幅，此处不再展开。"
- "这里不做过多的赘述。"
- "在此不做深入探讨。"
- "这个问题已经超出了本文的讨论范围。"
- "我们将在下一节中详细分析。"
- "正如我们前面提到的，"
- "后文会具体说明。"
- "此处不再一一列举。"
- "恕不赘述。"
- "因篇幅有限，本文仅作简要介绍。"
- "详细内容请参见……"

**替换策略**: 要么把内容写出来，要么别提起。对读者说"这里不展开"就是浪费读者时间。

> ❌ AI 写法: "限于篇幅，此处不再展开论述。"
> ✅ 人工写法: ——（删掉整句，读者根本不知道你曾经想展开什么）

## 被动/无主语

AI 不喜欢指责任何人，所以经常省略主语。找出来补上。

| 原文 | 改为 |
|------|------|
| 需要被重视 | 团队需要重视 |
| 可以被优化 | 我们可以优化 |
| 应该被关注 | 你应该关注 |
| 值得被讨论 | 我们来讨论 |
| 被寄予厚望 | 人们寄予厚望 |
| 得到广泛认可 | 行业认可了 |
| 被广泛应用 | 开发者正在用 |
| 被认为是 | 我认为是 / 行业认为是 |
| 被指出 | 批评者指出 |
| 被视作 | 很多人视作 |
| 被列为 | 政府列为了 |
| 被发现 | 研究者发现了 |
| 可以被实现 | 我们可以做到 |
| 需要被解决 | 我们必须解决 |
| 应当被考虑 | 你需要考虑 |
| 引起了广泛的关注 | 媒体关注了 |
| 获得了高度评价 | 用户给了好评 |

**替换策略**: 问自己"谁在做这件事？"，把答案写出来。

> **原则**: 中文不需要用被动语态来显得"客观"。主动句更短、更清晰、更像人写的。

## 总检查清单

写完文章后，用这个清单检查一遍：

- [ ] 有没有"废话开场"？（"值得注意的是……"）
- [ ] 有没有职场黑话？（"赋能"、"闭环"、"抓手"）
- [ ] 有没有"不只是……更是……"句式？
- [ ] 有没有"研究表明"之类的泛化引用？
- [ ] 有没有连续冒号或破折号滥用？
- [ ] 有没有空洞强调词？（"非常重要"、"极其关键"）
- [ ] 有没有假精确词？（"大量"、"显著提升"）
- [ ] 有没有三个成语连用？
- [ ] 有没有"随着……"开头段落？
- [ ] 有没有语气软化剂堆叠？（"可能大概也许"）
- [ ] 有没有"AI尾气"结尾？
- [ ] 有没有元评论？（"限于篇幅"）
- [ ] 有没有无主语句子？（"需要被重视"）

**每发现一个，删掉或重写。**
